Mr. Robøt vous permet de configurer les évènements à logger sur votre serveur Discord. Vous pouvez choisir parmi une liste d'évènements pour lesquels vous souhaitez recevoir des logs.

## Configuration
<Steps>
1. Ajouter [le robot sur votre serveur Discord](https://mrrobot.app/add)

2. Exécuter la commande
    ```
    /paramètres-logs
    ```

3. Choisir les évènements à logger dans le menu déroulant du panneau affiché
    ![Message de Mr. Robøt pour configurer les évènements à logger](/img/logs-add-remove-events.png)

4. Tester si vous le souhaitez la connexion en cliquant sur le bouton **Tester le webhook** du même panneau
</Steps>

## Les différents évènements

Vous avez une multitude d'évènements à votre disposition pour lesquels vous pouvez recevoir des logs. Voici la liste des évènements disponibles :
- **Insulte détectée** : Un message contient une insulte.
- **Spam détecté** : Un message ou une série de messages est détecté comme du spam par l'anti-spam.
- **Sourdine appliquée** : Un membre est mis en sourdine (timeout).
- **Membre expulsé** : Un membre est expulsé du serveur.
- **Membre banni** : Un membre est banni du serveur.
- **Membre débanni** : Un membre est débanni du serveur.
- **Honeypot déclenché** : Un membre écrit dans le [salon piège honeypot](/configuration/honeypot) et est banni automatiquement. Couvre aussi les changements de configuration du honeypot (salon choisi, création, activation/désactivation).
- **Membre rejoint** : Un membre rejoint votre serveur Discord.
- **Membre parti** : Un membre quitte votre serveur Discord.
- **Messages supprimés en masse** : Plusieurs messages sont supprimés en même temps.
- **Message supprimé** : Un message est supprimé sur votre serveur Discord.
- **Invitation créée** : Une invitation est créée.
- **Annonce créée** : Une annonce est créée via une commande de modération.

## Activer ou désactiver un évènement

Vous avez la possibilité d'activer ou de désactiver un évènement à logger dans le menu déroulant.

<Aside type="note" title="Légende">
  - ✅ L'évènement est activé.
  - ❌ L'évènement est désactivé.
</Aside>
<Steps>
  1. Exécuter la commande
      ```
      /paramètres-logs
      ```

  2. Activer les évènements que vous souhaitez logger en cliquant dans le menu déroulant :
      ![Message de Mr. Robøt pour configurer les évènements à logger](/img/logs/activateEvents.png)

  3. Fermez le menu déroulant pour enregistrer vos modifications.
  4. C'est enregistré ! 🎉
</Steps>

## Aucun évènement n'est loggé

Veuillez lire les étapes à suivre dans la partie [Mise en place des logs](/configuration/logs/#aucun-évènement-nest-loggé) pour configurer les évènements à logger.

## Permission

Permission par défaut : **Gérer le serveur**.
